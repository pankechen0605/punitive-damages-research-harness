# 层级契约

本文规定五层目录的输入、输出、边界和完成标准。任何 agent 或人工协作者在移动材料前，应先确认材料符合对应层级契约。

## 0_raw 原始材料层

### 这一层回答什么问题

原始材料是什么？

### 输入

- 法条、司法解释、规范性文件原文
- 指导案例、典型案例、人民法院案例库参考案例、公报案例、普通裁判文书原文
- CSSCI 论文 PDF
- PDF 转出的机器可读 `extracted_md`
- NotebookLM 导出的原文摘录或带来源定位片段

### 允许的处理

- 文件清理，例如去除网页广告、导航栏等噪音
- PDF 转 Markdown，前提是输出只保留可读文本，不加入模型总结
- 建立或更新 `source-index.md`
- 标记来源、日期、类型、是否已转 MD、是否已生成卡片

### 输出

- 可追溯的原始材料集合
- 机器可读副本
- 来源索引

### 禁止事项

- 不能放 AI 总结
- 不能放 MiMo 生成的 case card
- 不能放 NotebookLM / Gemini 的观点归纳
- 不能把人工判断写进原文文件
- 不能把未经来源核验的材料作为真实材料入库

### 进入下一层的条件

- 材料有明确文件路径
- 材料类型已经标注
- 能在 `source-index.md` 中找到或准备新增 Source ID
- 机器可读副本与原始 PDF 或原文保持对应关系

### 失败模式

- 把“学界认为”“法院通常认为”等 AI 归纳放入 raw
- 只有摘要，没有原文
- PDF 与 extracted_md 无法对应
- 来源信息缺失，后续无法核验

### 人工检查点

- 确认材料不是模型生成的总结
- 确认材料来源可信
- 确认文件放入正确子目录
- 确认 `source-index.md` 已更新

## 1_digest 单一材料消化层

### 这一层回答什么问题

单个材料说了什么？

### 输入

- `0_raw` 中的单个案例、法条、司法解释、论文或 NotebookLM 原文摘录
- 人工指定的单一材料处理任务

### 允许的处理

- 将单个案例整理成 case card
- 将单个规范整理成 rule card
- 将单篇论文整理成 paper card
- 将 NotebookLM 的中间输出暂存到 `notebook_outputs`
- 标记遗漏、矛盾、待核验项

### 输出

- `case_cards`
- `rule_cards`
- `paper_cards`
- `notebook_outputs`

### 禁止事项

- 不能做横向结论
- 不能写“本类案件法院通常……”
- 不能把论文观点写成法院规则
- 不能将一个案例的事实直接推广为法律结论
- 不能补充原材料没有的信息

### 进入下一层的条件

- 卡片能对应明确 source
- 关键字段完整或标记“待核验”
- 引用、事实、金额、案号、法院等信息已尽量从原文提取
- 不确定内容没有被写成确定结论

### 失败模式

- 案例卡像最终报告，而不是单案摘要
- 论文卡把作者观点写成裁判规则
- 规则卡没有区分条文、解释和裁判要点
- 未标记缺失字段

### 人工检查点

- 核对卡片是否忠于单一材料
- 核对请求权基础、主观要件、客观要件、赔偿基数是否遗漏
- 核对是否存在无来源推断

## 2_framework 跨材料观察框架层

### 这一层回答什么问题

我们应该用什么维度观察材料？

### 输入

- `1_digest/rule_cards`
- `1_digest/paper_cards`
- `1_digest/notebook_outputs`
- 课堂范围约束、老师提示、研究问题草稿
- 已有 case card 中暴露出的字段需求

### 允许的处理

- 建立规范地图
- 建立学术争议地图
- 设计裁判观察要素表
- 设计案例卡字段和横向比较字段
- 把论文争议转化为案例检索任务或观察问题

### 输出

- `normative_map`
- `literature_debate_map`
- `adjudication_factors`
- `field_design`

### 禁止事项

- 不能把论文争议写成法院规则
- 不能下最终法律结论
- 不能替代案例验证
- 不能把课堂提示包装成法律规则
- 不能为了理论完整而扩大当前 pre 范围

### 进入下一层的条件

- 观察字段可以被案例材料实际填充
- 每个争议能对应检索任务、案例字段或比较维度
- 规范锚点、论文观点、课堂范围约束已经区分清楚

### 失败模式

- 只有宏大理论，没有可操作字段
- 把学者争议直接写进规则总结
- 观察维度过多，无法服务当前 pre
- 案例比较时找不到对应字段

### 人工检查点

- 检查 framework 是否服务当前选题
- 检查是否需要删减过度理论化内容
- 检查是否把论文观点误当法院规则

## 3_synthesis 横向综合层

### 这一层回答什么问题

多个材料放在一起说明什么？

### 输入

- 经核验的 case cards
- rule cards
- paper cards
- `2_framework` 中的观察维度

### 允许的处理

- 横向比较支持与不支持惩罚性赔偿的案例
- 总结裁判因素
- 对比赔偿基数、倍数、责任竞合处理
- 形成文档版 pre 材料
- 标记尚不足以支持结论的问题

### 输出

- `cross_case_comparison`
- `rule_summary`
- `final_doc`

### 禁止事项

- 不能从少量案例过度概括
- 不能无来源形成法律结论
- 不能忽略反例或不支持案例
- 不能把展示语言替代研究判断

### 进入下一层的条件

- 横向结论能回溯到案例卡或规则卡
- 支持案例、不支持案例、权威锚点的关系已经说明
- 不确定问题已经标记

### 失败模式

- 案例数量不足却写成稳定规则
- 没有区分法院说理和作者评价
- 只选支持自己观点的案例
- 结论不能回到原始材料

### 人工检查点

- 检查结论支撑是否足够
- 检查是否有支持与不支持样本
- 检查规则是否来自法院而非论文

## 4_output 输出层

### 这一层回答什么问题

如何把研究结果交给组员或课堂展示？

### 输入

- `3_synthesis/final_doc`
- `3_synthesis/cross_case_comparison`
- `3_synthesis/rule_summary`
- 人工确定的课堂时间、组员分工、展示目标

### 允许的处理

- 生成 PPT 素材提纲
- 生成讲义
- 压缩表格为展示版
- 调整表达顺序和课堂节奏

### 输出

- `ppt_materials`
- `handout`

### 禁止事项

- 不能为了展示效果篡改研究结论
- 不能新增未经 `3_synthesis` 支撑的实质判断
- 不能反向污染 `3_synthesis`
- 不能把比较法引入扩展成新的主线，除非人工明确调整范围

### 进入下一层的条件

这是最终展示适配层，不再进入新的研究层。若输出中发现研究缺口，应回到对应层级补充，而不是在 output 中自行补结论。

### 失败模式

- PPT 素材比研究文档更“大胆”
- 为了讲得顺而删掉不确定性
- 把引入案例讲成比较法主线

### 人工检查点

- 检查 output 是否忠于 synthesis
- 检查是否适合课堂时间
- 检查是否保留必要不确定性
