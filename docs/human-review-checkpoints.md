# 人工核验检查点

## Raw 入库前

- [x] 是否为原始材料？
- [x] 是否有来源信息？
- [ ] 是否需要 source ID？
- [ ] 是否可能是 AI 总结？
- [x] 是否放入正确目录？
- [x] 是否需要保留 PDF 底本？
- [x] 是否需要生成 extracted_md？
- [ ] 是否已经更新 `source-index.md`？

## Digest 生成后

- [ ] 是否忠于单一材料？
- [ ] 是否把结论写成事实？
- [ ] 是否遗漏请求权基础？
- [ ] 是否遗漏主观要件？
- [ ] 是否遗漏客观要件？
- [ ] 是否遗漏赔偿基数？
- [ ] 是否遗漏判项或法院核心说理？
- [ ] 是否标记待核验？
- [ ] 是否存在无来源补充？

## Framework 形成后

- [ ] 是否把论文观点误当法院规则？
- [ ] 是否能转化为案例检索任务？
- [ ] 是否能转化为案例观察字段？
- [ ] 是否过度理论化？
- [ ] 是否服务当前 pre 范围？
- [ ] 是否区分规范锚点、论文观点和课堂约束？

## Synthesis 前

- [ ] 案例数量是否足以支撑横向比较？
- [ ] 是否有支持案例和不支持案例？
- [ ] 是否有权威案例或规则锚点？
- [ ] 是否有基数样本？
- [ ] 是否有倍数样本？
- [ ] 是否有责任竞合样本？
- [ ] 是否标记证据不足的问题？

## Output 前

- [ ] PPT 素材是否忠于最终文档？
- [ ] 是否过度简化？
- [ ] 是否遗漏不确定性？
- [ ] 是否新增未经 synthesis 支撑的结论？
- [ ] 是否为了展示效果改变研究判断？
- [ ] 是否仍服务当前 pre 范围？
