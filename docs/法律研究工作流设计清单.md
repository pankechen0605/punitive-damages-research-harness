# 法律研究工作流设计清单

## 定位

本清单用于启动新主题或重构现有主题时设计工作流。它借鉴法律工作流中 playbook、风险容忍度和人工复核门禁的思想，但只服务本仓库的研究管线。

## 一、使用对象

- 主要使用者：
- 协作 agent：
- 人工复核者：
- 最终读者：

检查问题：

- 这套流程是给课程 pre、论文写作、合规研究、案例检索还是内部决策准备使用？
- 输出会不会被外部依赖？如果会，人工复核门槛应提高。

## 二、研究形态

选择一种主形态：

- 案例分析
- 规范解释
- 学术争议整理
- 合规风险识别
- 跨案例比较
- 展示材料适配

不得在未说明研究形态时直接生成字段或结论。

## 三、AI 能做到哪一步

明确允许：

- 材料分类
- 单材料消化
- 研究对象提取
- 判断包草稿
- 风险提示
- 输出草稿组装

明确禁止：

- 编造材料
- 联网补充未经确认的内容
- 替人做最终法律判断
- 把未核验观点写成结论

## 四、人必须在哪一步接管

必须人工复核：

- raw 材料是否纳入。
- 规则来源是否有效。
- case card 是否忠于原文。
- factor model 和 weighting model 是否锁定。
- 判断包是否可升级。
- final legal conclusion 是否成立。
- output 是否改变研究结论。

## 五、输入材料

列明需要的材料：

- 规范文本：
- 案例材料：
- 论文材料：
- 机构材料：
- 数据或表格：
- 用户自有材料：

缺材料时不得用模型常识补齐，应写“待核验”或启动 backfill。

## 六、材料权威层级

建议顺序：

1. 法律、法规、司法解释、规范性文件原文。
2. 指导案例、典型案例、公报案例等权威案例。
3. 普通裁判文书。
4. 学术论文。
5. 机构报告、律所文章、公众号、新闻报道。

低权威材料可以提示问题、提供线索或解释背景，但不能单独支撑最终法律结论。

## 七、允许表述

为本主题写清楚哪些话可以说：

- 观察性表述：
- 趋势性表述：
- 待复核判断：
- 人工确认后可说的结论：

## 八、禁止表述

为本主题写清楚哪些话不能说：

- 缺少规范锚点时不能说：
- 缺少案例覆盖时不能说：
- 缺少数据时不能说：
- 没有 human sign-off 时不能说：

## 九、置信度和不确定性

每个判断包至少标记：

- 不确定性：高 / 中 / 低
- 原因：
- 需要补充的材料：
- 是否必须人工复核：

置信度不能替代来源，也不能替代人工法律判断。

## 十、输出格式

选择输出：

- case card
- rule card
- paper card
- 判断包
- 论证包
- 风险包
- 横向比较表
- final_doc
- PPT material outline

每种输出必须绑定所属层级和可用边界。

## 十一、流程可用性判断

流程可以使用的最低条件：

- 已有 topic config。
- 已说明材料权威层级。
- 已说明允许表述和禁止表述。
- 已说明人工复核点。
- 已说明输出格式。

流程不得进入批量生产的情形：

- 没有 topic config。
- 没有 factor model 或 weighting model。
- 没有 pilot。
- 判断包缺少来源或反向材料。
- 需要最终法律结论但没有 human sign-off。
