# 可追溯法律研究组装框架 v0.1

## 定位

本框架是本仓库的通用方法层，用于把不同法律或合规主题中的原始材料，拆解为可追溯的研究对象，再组装为待人工复核的判断包、论证包、报告段落、表格和展示材料。

它不替代既有五层目录：

```text
0_raw → 1_digest → 2_framework → 3_synthesis → 4_output
```

它规定五层目录中材料如何被拆、如何被绑定、如何形成判断、如何被校验。当前生态环境惩罚性赔偿只是一个 topic instance；同一方法可迁移到 BEPS、Temu/PDD 双支柱、数据合规、平台合规、游戏合规等主题。

## 核心抽象

```text
原始材料
→ 研究对象
→ 判断包
→ 论证与输出
→ 质量校验
→ 人工复核
```

研究对象包括来源卡、事实卡、规则卡、争点卡、裁判因素卡和证据卡。判断包用于汇集支持事实、规则依据、反向证据、限制因素、允许表述、禁止表述和下一步动作。输出层只能从已经过校验和人工确认的判断包、论证包或 synthesis 材料中组装表达。

## 与五层目录的关系

| 层级 | 框架功能 | 可放内容 | 禁止事项 |
|---|---|---|---|
| `0_raw` | 保存底本 | 原始材料、机器可读副本、来源索引 | AI 总结、判断包、横向比较、法律结论 |
| `1_digest` | 单一材料拆解 | case card、rule card、paper card、来源/事实/规则/证据对象草稿 | 横向结论、最终规则、无来源推断 |
| `2_framework` | 观察维度设计 | topic config、观察要素、权重、字段模型、争点模型 | 最终法律结论、凭直觉设高权重 |
| `3_synthesis` | 横向组装 | 判断包、论证包、风险包、比较表、claim ledger | 没有来源或 human sign-off 的最终法律结论 |
| `4_output` | 展示适配 | PPT 提纲、handout、展示表格 | 新增、强化或删改 synthesis 结论 |

## 基本工作方式

1. 先确定 topic config。任何新主题都必须先说明研究目的、材料边界、允许表述、禁止表述和人工复核点。
2. 再收集 raw 材料。`0_raw` 只放原始材料或不含模型总结的机器可读副本。
3. 再拆单一材料。`1_digest` 只回答单一材料说了什么，不做横向结论。
4. 再设计观察框架。`2_framework` 只回答读材料时看什么、优先看什么、如何记录。
5. 再 pilot。少量材料试跑用于检查字段是否可观察，不用于直接概括规则。
6. 再生成判断包。判断包必须绑定来源、事实、规则、反向材料、不确定性和禁止表述。
7. 再挑战判断包。检查来源权威、证据链、过强措辞、反向材料和人工复核点。
8. 最后组装输出。报告、表格和展示材料不得越过判断包和 human sign-off。

## 结论边界

AI 可以提取、整理、归类、比对、提出风险和生成待复核草稿。AI 不得自行完成最终法律判断。

允许表述示例：

- “材料显示存在支持该判断的事实与规则依据，需人工复核。”
- “当前证据链可支持观察性结论，但不足以升级为最终法律结论。”
- “该项为 backfill clue，尚不能作为 rule anchor。”

禁止表述类型：

- 在没有来源时写确定法律结论。
- 把论文观点写成法院规则。
- 把普通案例单独写成规则锚点。
- 把事实观察直接提升为最终法律判断。
- 在缺数据时写“必然”“显然”“已经确定”等强表述。

## PR1 边界

本框架 v0.1 只建立方法论、模板和提示词骨架。它不迁移旧案例，不更新已有 case cards，不新增实体法律结论，不修改 MCP server，不做 Temu 或 BEPS 实体研究。
