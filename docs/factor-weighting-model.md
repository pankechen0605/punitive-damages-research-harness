# 观察要素权重模型

## 一、什么是观察要素权重

观察要素权重是对某一研究主题下各观察要素的“研究优先级”判断。

它回答：

- 哪些要素每个案例都必须观察？
- 哪些要素必须进入横向比较？
- 哪些要素适合进入最终文档主线？
- 哪些要素只是背景或延伸？

它不回答：

- 法院最终规则是什么？
- 某个案件会不会支持某项请求？
- 哪个法律观点一定正确？

观察要素权重不是法律结论，不是预测模型，不是裁判结果评分。它只是本研究项目的注意力分配机制。

## 二、为什么需要权重

如果没有权重，项目容易出现：

- 文献讨论多的问题挤占真正重要的规范要件
- 案例卡字段过多，难以填写
- 横向比较表失控
- 最终文档主线发散
- AI 按论文热度而不是研究目标设计字段
- 看到新案例就不断改标准，导致移动靶心

权重的作用是让研究者在批量读案例之前，先明确哪些变量是核心，哪些变量只是辅助。

## 三、权重的五个来源维度

### 1. Normative Weight

问题：该要素是否来自法条、司法解释、指导案例或规范结构？

评分：0-3

说明：如果某要素直接来自规范结构或构成要件，通常具有较高基础性。但规范重要不等于自动进入展示主线，还要考虑案例可观察性和任务相关性。

### 2. Literature Weight

问题：该要素是否被论文高频讨论，是否存在明显学术争议？

评分：0-3

说明：论文热度不能单独决定最终优先级。论文只能提示研究问题，不能替代规范结构和案例验证。

### 3. Case Observability

问题：该要素是否能在裁判文书中稳定观察、摘录和填写？

评分：0-3

说明：如果案例里经常看不到，即使理论重要，也不适合设为 P0。此类要素可以保留为 P1/P2，或作为待进一步搜集材料的问题。

### 4. Discriminative Power

问题：该要素是否有助于区分支持案例、不支持案例、边界案例或不同裁判路径？

评分：0-3

说明：如果某要素能解释案例之间的差异，通常更适合进入横向比较。

### 5. Task Relevance

问题：该要素是否服务当前任务目标、课堂 pre、研究报告、作品集或展示主线？

评分：0-3

说明：同一主题在不同任务中可能有不同权重。课堂 pre 可能强调可讲清楚的主线，研究报告可能强调规范完整性，实证项目可能强调可稳定编码的字段。

## 四、Priority Level

- P0：核心要素。每个案例必填，横向比较必须出现，最终输出应当重点讨论。
- P1：重要要素。多数案例应尽量填写，通常支撑主线或重要分支。
- P2：辅助要素。有材料就填写，可作为补充分析或延伸。
- P3：背景要素。一般不进入主线，只作背景或后续研究线索。

Priority Level 不是简单机械加总，最终由人工根据五个维度综合决定。

## 五、权重状态机

1. `draft`
   - 刚从研究问题生成，未充分参考材料。

2. `literature-informed`
   - 已参考 paper cards 或 literature debate map。

3. `normative-adjusted`
   - 已对照 rule cards、normative map、法条或司法解释结构。

4. `pilot-tested`
   - 已用 2-3 个案例试跑，确认字段可填写。

5. `locked-v1`
   - 可以开始批量整理案例，暂时不随意改动。

6. `revised`
   - 根据后续案例样本进行版本化调整。

批量 case card 开始前，观察要素权重至少应达到 `pilot-tested`，最好达到 `locked-v1`。

## 六、如何避免 AI 编造

- 每个权重分数必须有 basis。
- basis 可以来自 rule card、paper card、topic config、pilot case note。
- 如果没有 basis，必须标记“待核验”。
- AI 不得凭直觉给高权重。
- AI 不得把高权重要素写成“法院通常认为”。
- 在 `locked-v1` 前，不得把观察要素权重当成最终研究结论。

## 七、权重如何影响下游

权重会影响：

- topic config
- adjudication factor model
- derived case fields
- case card 必填字段
- cross-case comparison 列
- rule summary 结构
- final doc 主线
- output 层展示重点

P0 要素：

- 进入 case card 必填字段
- 进入 cross-case comparison 主列
- 进入 synthesis 主体分析

P1 要素：

- 进入 case card 重要字段
- 进入 cross-case comparison 辅助列
- 视材料进入 synthesis

P2 / P3 要素：

- 不强制每个案例填写
- 可作为延伸、备注或后续研究线索
