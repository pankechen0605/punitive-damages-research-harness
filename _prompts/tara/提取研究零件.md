# 提取研究零件 Prompt

## 任务

基于用户提供的单一材料或已指定的仓库材料，提取可追溯研究零件。只使用提供材料，不得联网搜索，不得补充外部内容，不得生成最终法律结论。

## 输入

- 材料文本或文件路径：
- 来源编号：
- 研究主题：
- 适用 topic config：

## 输出对象

按需要生成：

- 来源卡
- 事实卡
- 规则卡
- 争点卡
- 裁判因素卡
- 证据卡

使用 `_templates/atoms/` 下的对应模板。

## 操作规则

1. 先识别材料类型和权威层级。
2. 只提取原文中可定位的信息。
3. 区分事实、规则、观点、主张、法院认定和背景材料。
4. 每个事实、规则或证据都必须绑定来源编号。
5. 不确定内容标记“待核验”。
6. 发现规范缺口时只写 backfill clue，不得直接写 rule anchor。

## 禁止

- 不得把论文观点写成法院规则。
- 不得把普通案例写成规则锚点。
- 不得从事实直接推出法律结论。
- 不得使用“必然、显然、确定、已经确立”等强表述。

## 结尾检查

输出末尾列出：

- 来源追溯是否完整。
- 权威层级是否清楚。
- 待核验项。
- 需要人工复核的字段。
