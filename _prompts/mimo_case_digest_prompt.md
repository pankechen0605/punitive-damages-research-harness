# MiMo 案例卡生成 Prompt

## 目标

根据我提供的**单个案例文本**，生成案例卡初稿。

## 严格要求

1. **只根据我提供的文本**，不补充任何外部信息。
2. **不编造案号、法院、裁判结果。** 如果文本中没有，留空。
3. **不联网搜索**，不补充任何我没有提供的案例、法条或司法解释。
4. 不确定的信息一律写 **"待核验"**。
5. 输出必须符合 `case_card_template.md` 的格式。

## 使用方式

请按以下格式提供输入：

```
【案例文本】
（粘贴案例全文或关键段落）
```

## 输出要求

请严格按照 `_templates/case_card_template.md` 的结构输出案例卡，逐项填写。

特别注意：
- **法律依据**：只写文本中明确提到的条文，不推测。
- **基本事实**：用 F-001、F-002 编号，只提取文本中的事实，不做法律评价。
- **法院判断**：严格区分法院认定的事实和法院的法律判断。
- **惩罚性赔偿裁量因素**：逐项检查，文本中没有提到的写"文本未提及"。
- **可提炼裁判规则**：只从法院的说理中提炼，不从学理中推导。
- **横向比较标签**：根据案例内容标注，用于后续横向比较。
- **待核验**：列出你不确定或需要人工核实的事项。
