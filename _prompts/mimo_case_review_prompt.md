# MiMo 案例卡审稿 Prompt

## 目标

检查案例卡是否存在以下问题，并给出修改建议。

## 检查清单

### 1. 事实与结论混淆
- 是否把法律结论写成了基本事实？
- 例如："被告故意侵权"是法律判断，不是事实。事实应该是"被告在收到警告后继续销售"。

### 2. 无来源推断
- 是否有文本中没有提到的信息被写入？
- 是否有推测性的法律依据？
- 是否有编造的案号、法院名称、裁判日期？

### 3. 主观要件遗漏
- 是否遗漏了法院对"故意/明知"的认定？
- 如果文本中没有提及主观要件，是否标注了"文本未提及"？

### 4. 客观严重性遗漏
- 是否遗漏了法院对"情节严重/严重后果"的判断？
- 是否遗漏了重复侵权、欺诈、拒不整改等情节？

### 5. 赔偿基数不清
- 赔偿基数的计算方式是否明确？
- 是否区分了实际损失、违法获利、许可使用费等不同基数？

### 6. 倍数裁量说明缺失
- 法院为什么选择该倍数？是否有说理？
- 如果文本中没有说理，是否标注了"文本未提及"？

### 7. 特别法与侵权责任编混淆
- 案例适用的是特别法（消费者法、食品安全法等）还是民法典侵权责任编？
- 是否被错误归类？

## 输出格式

```
## 审稿报告

### 整体评价
- 通过 / 需要修改 / 需要重写

### 问题列表

|---|---|---|---|---|
| R-001 | | | | |

### 修改建议摘要
-
```
