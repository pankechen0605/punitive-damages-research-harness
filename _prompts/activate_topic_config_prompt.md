# Activate Topic Config Prompt

用途：启动一个新的法律研究主题，并生成 `2_framework/field_design/current_topic_config.md` 草稿。

边界：

- 不生成法律结论。
- 不编造材料、案例、法条、论文或作者。
- 不把观察要素写成法院规则。
- 用户回答不足时，用“待确认”占位。
- 不一次性问太多问题。

## 使用方式

请按三轮提问执行。每轮等待用户回答后，再进入下一轮。不要跳过用户确认。

## 第一轮：研究边界

请只问以下问题：

1. 你的研究主题是什么？
2. 你最想回答的主问题是什么？
3. 这是用于什么任务场景？例如课堂 pre、论文、案例实证、memo、作品集等。
4. 这次明确不做什么？

如果用户回答不完整，用“待确认”占位，不要自行补全。

## 第二轮：材料和案例

基于第一轮回答，请只问以下问题：

1. 你预计需要哪些规范材料？例如法条、司法解释、规范性文件、权威案例等。
2. 你预计需要论文材料吗？如果需要，想让论文回答什么问题？
3. 你预计需要多少案例样本？
4. 你需要哪些案例类型？例如指导案例、典型案例、参考案例、公报案例、普通判决书、普通裁定书等。

如果用户回答不完整，用“待确认”占位，不要编造材料。

## 第三轮：输出和限制

基于前两轮回答，请只问以下问题：

1. 最终希望输出什么？例如 case cards、横向比较表、规则总结、final document、PPT 素材提纲等。
2. 读者是谁？例如老师、同学、组员、自己、面试官、内部团队等。
3. 有什么时间、篇幅或展示限制？
4. 有哪些人工核验点必须保留？

如果用户回答不完整，用“待确认”占位，不要自行补全。

## 生成 current_topic_config.md 草稿

完成三轮后，根据用户回答生成 `current_topic_config.md` 草稿，结构使用 `_templates/topic_config_template.md`。

生成要求：

- `type: topic_config`
- `location: 2_framework/field_design`
- `status: draft`
- `version: v0.1`
- 未确认内容写“待确认”
- 初步观察要素只能写成问题或变量，不能写成法院规则
- 不写具体案例，除非用户已经提供真实案例材料
- 不写法律结论
- 不编造材料来源

最后提醒用户：

```text
下一步应基于 current_topic_config.md 收集 raw 材料，并生成 rule cards / paper cards；在没有 topic config 前，不应直接生成观察要素模型、权重模型、case card 字段或横向比较列。
```
